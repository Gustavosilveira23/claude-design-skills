# shadcn/ui + Tailwind CSS Patterns

Reference for design system implementation patterns with shadcn/ui and
Tailwind CSS (v3 and v4). Read when you need implementation details.

---

## shadcn/ui Fundamentals

### Philosophy
- **Copy-paste, not npm package.** Components live in YOUR project.
- **Customizable by default.** You own the code, modify freely.
- **CSS variables for theming.** All colors come from CSS custom properties.
- **Radix UI primitives.** Accessible, unstyled building blocks underneath.
- **cn() utility.** Merges Tailwind classes with clsx + tailwind-merge.

### Project Structure

```
components/
  ui/                    # shadcn base components (auto-generated)
    button.tsx           # DON'T edit directly -- extend with wrappers
    card.tsx
    dialog.tsx
    ...
  [CustomComponent].tsx  # Your wrappers and custom components

lib/
  utils.ts               # cn() utility function

components.json          # shadcn configuration
```

## Tailwind CSS 4 vs 3

### Key Differences

| Feature | Tailwind CSS 3 | Tailwind CSS 4 |
|---------|---------------|---------------|
| Config | `tailwind.config.js/ts` | `@theme inline {}` in CSS |
| Import | `@tailwind base/components/utilities` | `@import "tailwindcss"` |
| CSS vars | Via `theme.extend.colors` | Via `@theme inline` block |
| Dark mode | `darkMode: 'class'` in config | Built-in `.dark` class support |
| PostCSS | `tailwindcss` plugin | `@tailwindcss/postcss` plugin |

### Tailwind CSS 4 Token Mapping

```css
@import "tailwindcss";

:root {
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
  --primary: oklch(0.6 0.2 30);
  --primary-foreground: oklch(1 0 0);
  /* ... all tokens */
}

.dark {
  --background: oklch(0.145 0 0);
  --foreground: oklch(0.985 0 0);
  /* ... dark mode tokens */
}

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --color-primary: var(--primary);
  --color-primary-foreground: var(--primary-foreground);
  /* This makes bg-primary, text-primary-foreground etc. work */

  --color-sidebar: var(--sidebar);
  --color-sidebar-foreground: var(--sidebar-foreground);
  /* Custom tokens also need mapping */

  --radius-sm: calc(var(--radius) - 4px);
  --radius-md: calc(var(--radius) - 2px);
  --radius-lg: var(--radius);
  --radius-xl: calc(var(--radius) + 4px);
}
```

### Tailwind CSS 3 Token Mapping

```js
// tailwind.config.js
module.exports = {
  darkMode: 'class',
  content: ['./app/**/*.{ts,tsx}', './components/**/*.{ts,tsx}'],
  theme: {
    extend: {
      colors: {
        background: 'var(--background)',
        foreground: 'var(--foreground)',
        primary: {
          DEFAULT: 'var(--primary)',
          foreground: 'var(--primary-foreground)',
        },
        // ... all tokens
      },
      borderRadius: {
        lg: 'var(--radius)',
        md: 'calc(var(--radius) - 2px)',
        sm: 'calc(var(--radius) - 4px)',
      },
    },
  },
}
```

## Component Patterns

### Extending shadcn Components

**DO: Wrapper pattern**

```tsx
// components/ActionButton.tsx
import { Button, type ButtonProps } from "@/components/ui/button"
import { cn } from "@/lib/utils"
import { Loader2 } from "lucide-react"

interface ActionButtonProps extends ButtonProps {
  loading?: boolean
  intent?: 'default' | 'success' | 'danger'
}

export function ActionButton({
  loading,
  intent = 'default',
  className,
  children,
  disabled,
  ...props
}: ActionButtonProps) {
  return (
    <Button
      className={cn(
        intent === 'success' && 'bg-success text-success-foreground hover:bg-success/90',
        intent === 'danger' && 'bg-destructive text-destructive-foreground hover:bg-destructive/90',
        className
      )}
      disabled={disabled || loading}
      {...props}
    >
      {loading && <Loader2 className="mr-2 h-4 w-4 animate-spin" />}
      {children}
    </Button>
  )
}
```

**DON'T: Editing components/ui/ directly**

The files in `components/ui/` are generated by `npx shadcn@latest add`.
If you edit them, your changes will be overwritten on the next add/update.
Always extend with wrappers in `components/`.

### Using CVA (Class Variance Authority)

For components with many variants:

```tsx
import { cva, type VariantProps } from "class-variance-authority"
import { cn } from "@/lib/utils"

const statusBadgeVariants = cva(
  "inline-flex items-center rounded-full px-2.5 py-0.5 text-xs font-medium",
  {
    variants: {
      status: {
        active: "bg-success/10 text-success border border-success/20",
        inactive: "bg-muted text-muted-foreground border border-border",
        pending: "bg-warning/10 text-warning border border-warning/20",
        error: "bg-destructive/10 text-destructive border border-destructive/20",
      },
      size: {
        sm: "text-[10px] px-1.5 py-0",
        md: "text-xs px-2.5 py-0.5",
        lg: "text-sm px-3 py-1",
      },
    },
    defaultVariants: {
      status: "active",
      size: "md",
    },
  }
)

interface StatusBadgeProps
  extends React.HTMLAttributes<HTMLSpanElement>,
    VariantProps<typeof statusBadgeVariants> {}

export function StatusBadge({
  status,
  size,
  className,
  ...props
}: StatusBadgeProps) {
  return (
    <span
      className={cn(statusBadgeVariants({ status, size }), className)}
      {...props}
    />
  )
}
```

### Composing shadcn Components

```tsx
// A search command palette built from shadcn primitives
import {
  CommandDialog,
  CommandEmpty,
  CommandGroup,
  CommandInput,
  CommandItem,
  CommandList,
} from "@/components/ui/command"

  return (
    <CommandDialog open={open} onOpenChange={onOpenChange}>
      <CommandInput placeholder="Search..." />
      <CommandList>
        <CommandEmpty>No results found.</CommandEmpty>
        <CommandGroup heading="Results">
          {items.map((item) => (
            <CommandItem key={item.id} onSelect={item.onSelect}>
              {item.label}
            </CommandItem>
          ))}
        </CommandGroup>
      </CommandList>
    </CommandDialog>
  )
}
```

## Token Naming Conventions

### Colors

Follow shadcn naming: `--[token]` and `--[token]-foreground`.

| Token | Purpose | Example |
|-------|---------|---------|
| `--background` | Page background | `bg-background` |
| `--foreground` | Default text | `text-foreground` |
| `--card` | Card/surface background | `bg-card` |
| `--card-foreground` | Text on cards | `text-card-foreground` |
| `--primary` | Brand/main action | `bg-primary` |
| `--primary-foreground` | Text on primary | `text-primary-foreground` |
| `--secondary` | Secondary surfaces | `bg-secondary` |
| `--muted` | Muted backgrounds | `bg-muted` |
| `--muted-foreground` | Secondary text | `text-muted-foreground` |
| `--accent` | Subtle highlights | `bg-accent` |
| `--destructive` | Error/danger | `bg-destructive` |
| `--border` | Default borders | `border-border` |
| `--input` | Input borders | `border-input` |
| `--ring` | Focus rings | `ring-ring` |

### Custom semantic tokens

Extend the system when needed:

```css
:root {
  /* Semantic feedback colors */
  --success: oklch(0.7 0.15 145);
  --success-foreground: oklch(1 0 0);
  --warning: oklch(0.75 0.15 75);
  --warning-foreground: oklch(0.2 0 0);
  --info: oklch(0.6 0.15 250);
  --info-foreground: oklch(1 0 0);
}
```

Then map in `@theme inline`:

```css
@theme inline {
  --color-success: var(--success);
  --color-success-foreground: var(--success-foreground);
  --color-warning: var(--warning);
  --color-warning-foreground: var(--warning-foreground);
  --color-info: var(--info);
  --color-info-foreground: var(--info-foreground);
}
```

Now you can use `bg-success`, `text-warning-foreground`, etc.

## Dark Mode Implementation

### CSS Variable approach (recommended)

```css
:root {
  --background: oklch(1 0 0);        /* white */
  --foreground: oklch(0.145 0 0);    /* near-black */
  --card: oklch(1 0 0);
  --primary: oklch(0.6 0.2 30);      /* saturated */
}

.dark {
  --background: oklch(0.145 0 0);    /* near-black */
  --foreground: oklch(0.985 0 0);    /* near-white */
  --card: oklch(0.2 0 0);            /* slightly lighter than bg */
  --primary: oklch(0.65 0.15 30);    /* slightly desaturated */
}
```

**Dark mode rules:**
- Background hierarchy: darkest furthest back, lighter forward
- Desaturate primary colors slightly (saturated colors vibrate on dark bg)
- Text: off-white, never pure #FFFFFF
- Borders: semi-transparent white (rgba(255,255,255,0.1))
- Elevation comes from lighter surfaces, not shadows
- Test at night in a dim room

### Theme toggle

```tsx
"use client"

import { useTheme } from "next-themes"
import { Button } from "@/components/ui/button"
import { Moon, Sun } from "lucide-react"

export function ThemeToggle() {
  const { theme, setTheme } = useTheme()
  return (
    <Button
      variant="ghost"
      size="icon"
      onClick={() => setTheme(theme === "dark" ? "light" : "dark")}
    >
      <Sun className="h-4 w-4 rotate-0 scale-100 transition-all dark:-rotate-90 dark:scale-0" />
      <Moon className="absolute h-4 w-4 rotate-90 scale-0 transition-all dark:rotate-0 dark:scale-100" />
    </Button>
  )
}
```

## Notes

- shadcn is NOT a component library you install from npm. It's a code
  generator that puts components in your project.
- `components/ui/` files are the generated base. Don't edit them directly.
- The `cn()` utility handles class merging correctly (tailwind-merge).
- All components use Radix UI underneath for accessibility.
- CSS variables are the theming layer. Change the variables, the whole
  system updates.
- When in doubt about a component, check https://ui.shadcn.com/docs
