# Psychology Principles by Situation

Principles organized by the situation the user is in. Read when designing
for a specific user context to select the most relevant psychological levers.

---

## When the user needs to choose between options
- **Hick's Law**: more options = harder decision. Reduce or group options.
- **Decoy Effect**: add a "decoy" option to make the desired one more attractive.
- **Centre-Stage Effect**: people tend to choose the middle option.
- **Framing**: how information is presented affects the decision (e.g., "90% success" vs "10% failure").
- **Anchoring Bias**: the first price/number the user sees becomes the reference.

## When the user might abandon mid-flow
- **Sunk Cost Effect**: users resist abandoning something they already invested in (time, data, money).
- **Goal Gradient Effect**: motivation increases as you approach the goal. Show progress.
- **Loss Aversion**: people prefer avoiding losses to gaining equivalents. Show what they lose by leaving.
- **Commitment & Consistency**: after a small action, users tend to maintain the behavior.
- **Pseudo-Set Framing**: tasks that are part of a group are more tempting to complete.

## When the user needs to trust the interface
- **Social Proof**: users adapt behavior based on what others do (reviews, numbers).
- **Authority Bias**: opinions from authorities carry more weight (logos, certifications, experts).
- **Aesthetic-Usability Effect**: beautiful interfaces are perceived as easier to use and more trustworthy.
- **Halo Effect**: a positive impression of one aspect influences overall judgment.
- **Noble Edge Effect**: users prefer socially responsible companies.

## When the user is overloaded with information
- **Cognitive Load**: reduce the mental effort needed to complete the task.
- **Progressive Disclosure**: reveal complexity gradually, not all at once.
- **Miller's Law**: working memory holds 5 plus or minus 2 items. Group information.
- **Chunking**: grouped information is easier to process and remember.
- **Tesler's Law**: over-simplifying can transfer complexity to the user.

## When the user needs to act now
- **Scarcity**: limited things are perceived as more valuable (stock, time, spots).
- **Nudge**: subtle cues can influence decisions without forcing.
- **Spark Effect**: users act more when perceived effort is small.
- **Fresh Start Effect**: sense of new beginning motivates action (Monday, new month, new year).
- **External Trigger**: when information about what to do is on the screen itself.

## When the user needs to remember or return
- **Peak-End Rule**: experiences are judged by the emotional peak and the ending.
- **Serial Position Effect**: first and last items in a list are remembered most.
- **Recognition Over Recall**: it's easier to recognize than remember. Show options instead of free input.
- **Zeigarnik Effect**: incomplete tasks are remembered more than complete ones. Use as a return hook.
- **Spacing Effect**: spaced learning is more effective than concentrated.

## When the user is discovering the product for the first time
- **Familiarity Bias**: people prefer what they already know. Use familiar UI patterns.
- **Mental Model**: users have expectations of how things work. Align or re-educate carefully.
- **Aha! Moment**: the moment the user realizes the product's value. Accelerate this moment.
- **Feedforward**: when the user knows what to expect before acting, the experience is smoother.
- **Skeuomorphism**: elements that resemble real-world objects ease adaptation.
- **Empty State Design**: empty states (no items, initial screen) are opportunities to guide. Use to guide, explain, or invite the first action -- never leave "nothing here yet."
- **Safe Exploration**: the user should be able to explore without fear of irreversible consequences. Back buttons, undo, and confirmations before destructive actions encourage exploration.

## When the user interacts repeatedly
- **Variable Reward**: unexpected rewards are more engaging than predictable ones.
- **Investment Loops**: the more the user invests, the more likely they return.
- **Flow State**: complete immersion in a task. Eliminate unnecessary interruptions.
- **Default Bias**: users tend to keep the default. Choose good defaults.
- **Internal Trigger**: when the user is motivated to act by an internal memory or emotion.
- **Streamlined Repetition**: if the user repeats the same action frequently, create shortcuts, automations, or pre-fills. Fewer clicks for recurring actions = more satisfaction.

## When the user needs to fill forms or provide data
- **Fitts's Law**: larger and closer targets are easier to interact with.
- **Feedback Loop**: show the result of each action immediately.
- **Signifiers**: elements should communicate what they do (inputs look editable, buttons look clickable).
- **Banner Blindness**: users ignore things that look like ads or that they see repeatedly.
- **Decision Fatigue**: many consecutive decisions degrade choice quality. Break into stages.
- **Deferred Choices**: not all information needs to be filled now. Allow users to skip non-essential fields and return later. The less mandatory at the moment, the lower the friction.
